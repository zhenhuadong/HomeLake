做JAVA 相关的可以讲一下ss sync这个项目：
2.1. 总体: 从产品设计，开发，集成，后期定制化开发，全流程参与一个产品。项目人数30多人，自己是key developer, DM leader. 
2.2. 问题：产品解决的是一个很小的问题， 呼叫转移在2G网络和4G网络的同步。 呼叫转移的信息，在2G和4G分别存在于不同的数据库，当用户在2G网络中，数据更新存在2G数据库。
当用户在4G网络中，数据更新存在4G数据库。问题是，用户在手机上更改自己的呼叫转移信息，希望在2G和4G中都生效。 3GPP没有定义这一同步细节，所以E///使用UPG这个工具来实现这一功能。
2.3. 框架: UPG运行在分布集群上，标准是2控制板+2业务板，使用glassfish作为server, zookeeper作为分布式配置管理和分布式锁，使用spring的bean生命周期来整合服务。UPG氛围的inbound接受请求 + core 管理线程和通用的pre-post处理 + JDV 业务逻辑 + outbound连接到电信内的各种数据源。
2.4. 我做了什么，inbound写了一个inbound的proxy, core JDV。 做了性能测试和调优。 
2.5. 使用了什么技术，inbound proxy主要是把不同inbound的来的请求统一处理，使用了分布式锁，并提供同步和异步两种处理模式。 JDV动态部署，需要动态的绑定业务处理逻辑和前端的请求(trigger). 同时要分别调用读取+比较+写入逻辑。 
2.6. 我遇到的最棘手的问题是什么？
多线程带来的好多个同步成功却使用了相同的号码。 由此展开线程池 + 锁 + synchronized 关键字，分布式锁
